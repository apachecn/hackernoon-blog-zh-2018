真正的“数据泄露年”请站出来好吗？

原文：https://medium.com/hackernoon/will-the-real-year-of-the-data-breach-please-stand-up-744ab6f63615?source=collection_archive---------46-----------------------#2018-01-04

我的元旦仪式已经持续了近10年:一份迟到的早餐，一杯浓咖啡，浏览安全博客和新闻，这两件事总是让我发笑:新年的网络预测，以及宣布过去一年为“数据泄露年”的回顾会。凯利·肖特里奇完美地模仿了前者，我实际上认为我们可能会有一年没有后者，直到我在彭博新闻上发现了这个标题，其中 2017年被命名为数据泄露之年:

如果你想知道，我以前在哪里见过这个…？每年都快了。准确地说，是过去12年中的10年。

这里是对过去12年的回顾和对2018年前景的一瞥。

2005

2005年是被许多媒体宣布为“数据泄露年”的第一年，比如信息世界。“数据泄露”一词也在2005年进入日常使用，由于数据泄露通知法的颁布，数据泄露诉讼增加，以及 Ameritrade 、花旗集团和 CardSystems 都披露了事故。

2006

2006年是数据泄露的重要一年——主要是美国在线搜索数据泄露丑闻和T2美国退伍军人事务部数据泄露事件。这使得一个博客作者和一个安全供应商将2006年称为数据泄露年。

2007

Attrition.org、身份盗窃资源中心和加拿大联邦隐私专员在给议会的一封信中都宣布2007年为“数据泄露年”我记得2007年有两件事:布兰妮的悲伤崩溃和 TJ Maxx数据泄露。

2008

没什么！2008年不是数据泄露年！干得好，2008。

2009

根据福布斯和安全厂商的说法，如果2005、2006和2007年都是数据泄露年，那么2009年就是大规模数据泄露年。这是一个大问题，主要是由于心脏地带支付系统的数据泄露导致1.3亿条记录受损。

2010

在经历了2009年的辉煌之后，我们都决定休息一下。

2011

在2008年和2010年不是数据泄露年之后，就好像安全记者、供应商和网络专家都站起来，异口同声地喊道，“再也不会了！永远不会有不是数据泄露的一年！”

那是一个好年景。趋势科技称之为，Brian Krebs和其他许多人引用了它。最引人注目的事件是索尼Playstation Network 遭遇长时间服务中断和数据泄露。

2012

一家小型安全厂商在年终回顾中，将2012年命名为“数据泄露年”，雅虎、 Zappos 和政府部门的几起高调事件占据了新闻头条。根据新西兰隐私专员的说法，这也是“新西兰数据泄露年”。

2013

2009想要回它的形容词。赛门铁克在《2013年互联网安全威胁报告中称2013年为“大规模数据泄露年”，列举了针对中小型企业和政府部门的攻击。由于发生在塔吉特和土坯的事件，其他人称为“零售商违约年”。

2014

假设我们只能有一个“数据泄露年”，2014年将是最强有力的竞争者。2014年发生了大量的事件:联合包裹、迈克尔、家得宝、吉米·约翰、史泰博和摩根大通。前面提到的都黯然失色，世纪黑客(根据财富):索尼电影娱乐黑客。

大多数媒体将2014年称为“数据泄露年”，还有 Advisen 、趋势科技和Tripwire的安全状态。

2015

我宣布2015年是“最棒的一年”。这一年是如何报道的:

  • 《财富》杂志将2015年称为“数据泄露诉讼年”
  • 安全厂商IDM365称之为“超级大入侵年
  • 趋势科技称之为普通的“数据泄露年”然而，趋势科技也宣布2014年为数据泄露年。
  • Vice.com称2015年为“医疗违规年”
  • Hacked.com称之为“个人数据泄露年
  • GovTech.com选定了“数据泄露成为隐私的一年”
  • HIPAA Journal将2015年称为“大型医疗数据泄露的年”

2015年，许多美国人受到数据泄露的影响，最引人注目的事件发生在阿什利·麦迪森、 OPM 、国歌和国税局。

2016

2014年和2015年之后，人们可能会认为是时候让“年”这个词退休了，换个说法。没有。

一家专门从事数据隐私的小型律师事务所、 SecureWorld 和电台主持人将2016年命名为“数据泄露年”。

在一份完全令人费解的声明中，Ars Technica将2014年和2015年视为“数据泄露年”，并且还发布了一项挑战:

如果专家们不给[2016]贴上“数据泄露年”的标签，就像21世纪的中国黄道十二宫一样，他们根本没有做好自己的工作。

2017

彭博宣布2017 为“数据泄露年”，列举了发生在 Equifax 、雅虎和优步的事件。益百利也加入了这股潮流。

2018:网络预测

结合我最喜欢的两件事:网络预测和“数据泄露年”宣言，非营利组织信息安全论坛 (ISF) 称2018年将是“数据泄露年”。

结论

关于消费者数据泄露疲劳已经写了很多。我毫不怀疑违约疲劳是真实存在的，年复一年，类似这样的头条新闻会助长这种疲劳。当关于网络安全的头条新闻过分夸张时，是时候重新思考我们如何向世界其他地方呈现该行业最紧迫的问题了。目前，宣布一年为“数据泄露年”已经变得毫无意义。我们知道数据泄露每年都会发生。或许，从今年开始，我们可以将一次值得注意的数据泄露称为“年度数据泄露”，而不是将全年命名为“数据泄露年”
