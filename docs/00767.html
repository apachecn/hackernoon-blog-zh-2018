宣布推出Exoframe 2.0 —在几分钟内部署任何东西

原文：https://medium.com/hackernoon/announcing-exoframe-2-0-deploy-anything-within-minutes-264fdcc4d6?source=collection_archive---------27-----------------------#2018-01-24

Exoframe是一个自托管工具，允许使用Docker进行简单的单命令部署。
如果你不熟悉，你可以在关于初始发布的文章和项目库中找到更多信息。

我很高兴地宣布 Exoframe 2.0的发布！
该版本包括社区要求的许多附加功能，以及将Exoframe服务器迁移到 Fastify 和测试到 Jest 。

项目模板

2.0版本的主要亮点是支持第三方模板。
有了这个，Exoframe不再局限于部署Node.js、静态HTML、Docker和docker-compose项目。现在可以部署任何类型的项目——只要您安装了正确的模板！
模板将自动检测您正在部署的项目类型，或者可以通过Exoframe配置文件明确设置。

除了对模板的支持，我还将为 Maven 、 Java 和 Tomcat 项目发布三个基本模板。

你可以在文档中阅读更多关于模板的内容。如果你对创建自己的模板感兴趣，看看这个指南。

带有Letsencrypt挑战修复程序的Traefik 1.5

另一个主要变化是对Traefik 1.5.0的更新，增加了对ACME HTTP质询的支持，这是letsencrypt正确工作所必需的，因为TLS-SNI-0x质询已被禁用。

更多好吃的

其他较小(但很棒)的新增功能包括:

  • 现在，您可以通过Exoframe config设置附加容器标签(当您想要传递附加Traefik参数时非常有用)
  • 服务器现在可以通过配置启用可选的 CORS支持
  • 现在可以配置与Exoframe服务器一起使用的Traefik实例(例如，您可以将其与您自己的Traefik实例一起使用)
  • 对于那些愿意尝试最新版本的人来说，现在有了一种针对服务器和CLI更新到夜间版本的方法

我很想听到你对这个新版本的反馈。请随意将它发送到GitHub或Twitter！
