什么是渐进式网络应用程序(PWA)？

简单来说，渐进式web应用程序是移动和Web应用程序的混合物！

但是这到底意味着什么呢？这些与传统的Web应用和原生应用有什么不同？

  • pwa是渐进式的 —这些应用适用于每一个用户，不管你使用的是什么浏览器，甚至不管你在世界的哪个角落！所以不管你是使用Chrome还是Opera，也不管你是生活在印度、英国甚至是朝鲜，都没关系！渐进式Web应用程序也能很好地工作，因为它们是以渐进式改进为核心原则构建的。
  • 响应式 —先进的网络应用将适用于任何设备！无论是台式机、手机、平板电脑，甚至是还没有被创造出来的东西！
  • 不依赖连接的网络应用 —在服务人员的帮助下，进步的网络应用甚至可以在低质量的网络上工作，甚至离线！
  • 类似应用的感觉 —渐进式网络应用感觉就像一个应用。他们有应用风格的交互和导航。
  • 新鲜 —感谢服务人员，您的Progressive Web应用程序将始终保持最新更新！
  • 安全 —渐进式网络应用程序始终通过HTTPS提供，确保没有人未经适当授权就能篡改你的应用程序。
  • 可发现的 —根据W3C清单，渐进式网络应用被归类为应用。由于服务人员注册范围允许搜索引擎轻松找到他们，因此更容易找到渐进式网络应用程序。
  • 用户参与 —推送通知等PWA功能让用户参与变得非常容易！
  • 安装——用户可以在自己的主屏幕上“保留”最有用的pwa，而没有应用商店的麻烦。
  • 可链接 — PWAs可以通过URL轻松共享，不需要复杂的安装。

本地应用与传统Web应用和PWAs

让我们对本地应用、传统Web应用和PWAs做一个简单的比较

原生应用

原生移动应用程序是一种智能手机应用程序，它是用像iOS的Objective-C或Android操作系统的Java这样的编程语言编写的。

原生应用具有以下特性:

  • 快速的性能和高度的可靠性。
  • 完全访问手机的硬件。
  • 原生应用甚至可以修改系统设置。
  • 本地应用程序甚至可以访问手机的各种设备和功能，如摄像头和通讯录。它还可以使用手机的电话功能发送信息或拨打电话。
  • 原生应用迭代较慢。这意味着用户需要一段时间才能获得更新。这是因为原生应用程序通常在应用程序商店中找到，所以开发者发送的任何更新都必须经过商店的监管才能提供给公众。

传统网络应用

传统的Web应用程序具有以下特征:

  • 实时交付 —根据你的网速，网络应用程序会立即交付给你。
  • Web应用程序是使用标准的 HTML 、 CSS 和 JavaScript 构建的。
  • 对设备功能的有限访问。因此，虽然你的应用程序可以访问相机或定位服务等功能，但它无法访问你的手机的电话功能。
  • 网络应用依赖于互联网连接。如果没有适当的连接，你所能做的就是玩Chrome恐龙。
  • 您无法在您的设备上安装web应用。
  • 任何更新都会被即时发送给所有用户。与本地应用不同，网络应用不需要考虑任何应用商店的限制。

渐进式网络应用(PWAs)

除了我上面提到的功能之外，pwa还由以下功能定义:

  • 低摩擦分布 — PWA更新即时！所有用户将立即同时得到提示！
  • 就像传统的网络应用一样，PWAs也将对设备的功能进行有限的访问。
  • pwa的一个主要缺点是 iOS尚未向pwa提供支持。但是苹果公司的开发人员似乎已经开始提供支持。

PWA中的可发现性

根据康姆斯克移动应用报告，超过50%的美国智能手机用户每月下载零应用！这意味着什么，我们的手机曾经充满应用的时代正在慢慢消逝！

据统计，下载一个应用程序的每一步都会减少该应用程序20%的潜在用户数量。

渐进式网络应用程序有效地减少了发现应用程序和在主屏幕上安装应用程序之间的步骤，从而消除了在设备上安装应用程序的摩擦。

PWAs的另一个胜利是，原生应用程序只能通过Play Store/App Store获得。这些商店成功的原因是用户在如何找到和安装原生应用程序方面没有其他选择。

但是说到网络应用(传统的和进步的)，没有单一的商店！因此，我们可以精心策划完全符合我们需求的应用商店。机会是无限的！

安装原生应用程序的一般流程如下:

  1. 打开App Store，搜索应用程序。
  2. 单击安装。
  3. 接受各种权限
  4. 打开应用程序
  5. 应用程序启动。
  6. 登录应用程序
  7. 使用应用程序

让我们将其与安装渐进式网络应用程序进行比较:

  1. 访问网站。
  2. 添加到主屏幕。
  3. 转到主屏幕并打开网站。
  4. 用app！

推送通知

推送通知是在用户设备上弹出的消息。推送通知可以由打开的应用程序在本地触发，也可以从服务器“推送”给用户，即使应用程序没有运行。推送通知允许用户选择及时更新，也允许您有效地重新参与定制内容。

推送通知使用两个API组装而成:通知API 和推送API 。通知API让应用程序向用户显示系统通知。推送API允许服务人员处理来自服务器的推送消息，即使在应用程序不活动时也是如此。

通知和推送API构建在服务工作器API 之上，后者在后台响应推送消息事件，并将它们转发给应用程序。

服务人员

我在这篇文章中多次提到服务人员。但是这些服务人员是什么呢？

服务人员是您的浏览器在后台运行的脚本，独立于网页/web应用程序。它通常用于不需要网页或用户交互的功能。

服务人员提供了离线体验、定期后台同步和推送通知等功能所依赖的技术基础。

如何推送通知？

要实现推送通知，您需要配置PWA的客户端和服务器端。

先从服务器端说起。

服务器端

在PWA的服务器端，

  1. 首先生成要发送给用户的数据。
  2. 然后使用用户公钥对日期进行加密。
  3. 服务器现在将加密数据发送到端点URL。

无生气的

服务器可以通过多种方式向用户发送推送通知。一种这样的方法是VAPID(用于网络推送的自愿应用服务器标识)协议。VAPID使用JSON Web令牌来携带信息。

索然无味的核心叫索赔。声明是包含几个公共字段的JSON对象。

应用服务器创建公钥/私钥对。公钥给你的web应用，而私钥由管理员保管。

当用户选择接收推送通知时，PWA将公钥添加到subscribe()调用的options对象中。

现在，应用程序的服务器发送一个push消息，它包括一个签名的JSON web令牌和公钥。

如果这个公钥与用户拥有的公钥相匹配，则数据被解密。

这种方法可以防止未经授权的推送服务，否则会导致更大的DOS攻击风险。此外，任何拥有端点的应用服务器都可以向您的用户发送消息，如果出现问题，推送服务无法联系开发人员。

离线功能

在旅途中，互联网连接经常会变得不稳定，甚至不存在。这就是PWA具有离线功能的原因，这使得它即使在完全没有互联网连接的情况下也能工作！

在PWA中，服务工作者充当网络代理。因此，当用户打开PWA的新屏幕时，对该页面的请求首先到达服务人员。然后，服务人员尝试从缓存中访问该内容。

如果服务人员可以提供内容，那么不管网络连接如何，我们都将获得新页面。否则，服务工作者将请求重定向到网络。

你应该什么时候更新你的PWA？

由开发人员来发布PWA的更新。开发人员可以随时发布更新。pwa不必为play store的限制而烦恼。

用户可以决定是否要更新他们的应用程序。因为服务人员首先将请求发送到缓存，所以应用程序的工作不会受到可用的新更新的影响。只有当应用程序关闭时，缓存才会更新。

开发人员需要确保新的更新不会干扰用户的交互性。所以建议开发者只要给用户发个提示更新app就行了，用户有空随时可以更新。

如果你曾经使用过Whatsapp Web，你可能会偶尔看到推送通知，通知你有新的更新。

带反应堆的PWA

有人可能想知道pwa与使用像 React 这样的库的现有架构有多兼容。PWA需要大规模重写吗？

如果使用React构建PWA，会有很多好处。这些优点之一是更好可发现性。

如果你开发一个单页面应用程序(SPA ),你将拥有客户端JavaScript，它将在浏览器中执行，然后谷歌将能够索引它。

React提供了方法React.renderToString()，该方法将React元素呈现为其初始HTML。您可以使用这种方法在服务器上生成HTML，并在初始请求时发送标记，以加快页面加载速度，并允许搜索引擎出于SEO目的抓取您的页面。

对PWA使用React的另一个优点是异步导入。
