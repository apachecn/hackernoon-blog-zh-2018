你可能没听说过的HTTP状态码

在野外可能不常见到。

我的一个同事偶然发现了一个团队不确定的 HTTP 状态代码: 207 。当我看的时候，我无意中发现了其他一些我在4年的开发生涯中从未真正意识到的东西。

让我们来看一些有趣的代码…

207:多状态

在适当的情况下，发送一个状态代码来表示有关多个资源的信息。

假设您有一个聚合层，它可能调用多个API来整理和处理数据，并返回一个 JSON 有效负载。如果其中一个调用成功，而其他几个调用失败，API该怎么办？

不是4xx，资源找到了；不是300，没有重定向；它不是5xx，聚合服务器没有处理失败并爆炸，所以在应用服务器的眼中，它在技术上是200。

207聪明；如果实现得当，它将为您提供一个XML有效负载，其中描述了关于分层调用的成功或失败的信息——每个元素都必须包含一个href来标识资源。接下来会发生什么取决于消费者。

411:所需长度

当服务器拒绝接受没有明确定义内容长度的请求时。

简单而有意义，但不是你可能看到的最常见的代码。

451:由于法律原因不可用

由于合法要求而发送的拒绝访问资源的状态代码。

451回复机构应包括对法律要求、提出要求的一方、适用的法律或法规以及适用于哪类人员和资源的解释。

有趣的是，451并不一定意味着被请求的资源存在——如果拒绝它背后的法律推理被删除，资源可能仍然不存在。

相当酷。

我是一个茶壶

是的，你没看错。此客户端错误响应表明服务器拒绝冲泡咖啡，因为它是一个茶壶。

别紧张，这是1998年的愚人节玩笑。

在官方征求意见上阅读更多信息。

506:变体也协商

506可用于说明服务器存在内部配置错误，其中所选资源被配置为参与透明内容调用本身，因此它不是协商过程中的适当端点。

实验性。

508:检测到循环

当服务器在处理请求时遇到无限循环时发生，发送此消息表示操作失败。

我个人并不经常看到这种情况，因为我发现当人们在代码中发现无限循环时，他们会修复它们，或者他们没有被发现而导致超时。
