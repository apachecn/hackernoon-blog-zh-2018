Git如何改变软件版本控制的历史

计算机出现的时间不长，但是已经发生了很多事情。一路走来，有几个不容错过的里程碑——想想互联网的出现、智能手机的商业成功以及当前无处不在的社交媒体。

然而，一些进展发生在远离聚光灯的地方。这些东西可能不会引起大众的兴趣，但它们对塑造我们今天使用技术的方式起到了重要作用。

软件开发可能是技术中最容易受到变化影响的单一事物。每隔几年，似乎就有一种新的、时髦的编程语言、框架或概念吸引了成千上万的开发人员蜂拥而至。当然，其中只有几个真正坚持下来了，Git无疑是其中之一。

今天，它是最常用的版本控制系统，它的影响力不容小觑。但是在我们进入Git之前，理解版本控制的概念是如何产生的，以及为什么它对软件开发的当前状态如此重要是至关重要的。

VCS:简史

计算机已经发展成为非常友好的机器。如果你搞砸了，并且不小心删除了一份重要文件中的几个段落，没有理由惊慌失措——你只需要点击撤销。但并不是一直都是这样。

早在数字革命的早期，大多数人甚至都没有听说过计算机——但是那些听说过的人使用起来却困难得多。当时的软件开发人员无法使用版本控制系统，而是与一个拥有项目主版本的人合作。这个人将只分享某个人需要工作的代码的特定部分，开发人员将返回完成的代码，最后在旧版本被新版本完全取代之前，所有内容都被检查是否符合标准。

总而言之，这是一个相当乏味的过程。除非有人主动制作重复副本，否则很难访问以前的版本。直到1972年的一天，开发者马克·罗奇金德再也处理不下去了。他最终创建了源代码控制系统(SCCS) ，第一个版本控制系统。它的功能非常有限——一次只能有一个人处理一个文件，并发开发是用锁来管理的。然而，它也解决了很多问题——多个版本不再需要占用大量的磁盘空间，合并优化变得更加容易。

第二代VCS由像 CVS 这样的工具组成，并且进行了全面的改进。总的来说，它们给了用户更多的灵活性，但是有一点仍然存在——在提交之前，需要合并修改。是时候解决这个问题并将版本控制带入一个新时代了。

Git:一颗冉冉升起的新星

不久之后，该行业发现自己处于第三代风投的黎明，这是第一次将合并和承诺分开。这些所谓的分布式版本控制系统旨在提高速度、数据完整性并支持分布式非线性工作流程。

在这波系统浪潮的最前沿是T4的Git T5，它是由Linux的共同创造者Linus Torvalds发明的DVCS。当它在2005年首次发布时，大多数人仍在使用 Apache Subversion (SVN) ，但在接下来的十年中，这两者陷入了某种形式的浏览器战争。今天，Git已经成长为领先的版本控制工具。即使是最受欢迎的第三代竞争对手 Mercurial ，在使用方面也远远落后于。

为什么Git很棒

那么Git为什么会被如此广泛的采用，又是如何成长为今天开发者的首选呢？主要是因为这是版本控制的巨大进步。

1.Git是分布式的

这是Git和它之前的所有东西之间最大也是最重要的区别。人们不再依赖中央服务器，而是能够在本地实时提交、记录和做任何其他事情，而无需使用VPN，甚至无需在线。突然之间，人们远程合作变得容易多了——甚至可以跨越时区。这还意味着，在紧急情况下，源数据不会像过去那样丢失，因为每个人在本地驱动器上都有一份完整的拷贝，他们可以将拷贝推送到服务器。

此外，开发人员现在可以访问更加灵活的工作流。Git可以与多个存储库一起工作，并在它们之间同步更改，这打开了一个可能性的世界。例如，您可以限制某人对特定存储库的写访问，这样经理就可以在将工作合并到中央存储库之前检查他们的工作

2.Git重新定义了分支和合并

虽然第二代风投公司中确实存在分支机构，但它们的工作方式让它们变得更加繁琐。在Git中，它们是系统的关键元素，可以完美地工作——例如，创建一个分支可以在本地立即进行。取决于你的互联网连接，服务器的状态和其他因素，在像Subversion这样的集中管理的VCS中，这可能需要更长的时间。这种快速的工作方式使您可以轻松地为您的任何想法创建快速分支，这样您就可以在以后决定是否要将它们添加到母版中或将其丢弃。

Git处理合并也比以前的系统优雅得多。因为它使用了不同的方式来记录提交历史,所以它能够记住已经合并的所有内容，从而大大减少了发生冲突的次数。

3.Git有一个中转区

Git的另一个很棒的特性是让生活变得更加简单。以前的VCS要求用户同时提交所有的变更，而staging area可以将所有的工作分成多个变更集。当这些都是分阶段的，并且是单独提交的，它会让其他人更快地找出你一直在做的事情的结构。

4.Git有最好的营销活动

尽管它本身就是一个出色的系统，但很难否认Git还有其他优点。事实上，它是由Torvalds创建的，Torvalds是一个有影响力的开发人员，拥有大量的追随者，这无疑有助于它的采用率——但还有一件事对成功有很大的贡献。

GitHub:隐藏的马达

当谈到Git时，不可能跳过GitHub——这是该系统受欢迎的驱动力之一，也是其本身对技术的一大影响。

在2008年推出时， Sourceforge 是开源开发者共享项目的默认方式——但是没有人真正喜欢它。这项服务充斥着广告，而且通常提供糟糕的用户体验。相比之下，GitHub就像一股新鲜空气——主要是因为它确实很好。当该公司宣布为开源代码托管将是免费的，每个人都开始使用它来进行他们的个人项目，随着时间的推移，这种流行渗透到了专业行业。

它很快成为Git的首选托管服务，提供了所有的标准特性，并在此基础上增加了新的特性。它有效地将DCVS变成了一个社交网络，增加了用户档案、公共存储库和拉式请求等功能，创建了一个独特的程序员社区。最终，它成长为世界领先的代码共享平台，数百万开发人员在这里就各种项目进行合作。

多年来，Git和GitHub从彼此的成长中获益匪浅。当与新程序员交谈时，这一点变得显而易见，他们经常混淆它们或者认为它们是同一件事。然而，最重要的是，它们的结合是软件开发的巨大进步，简化并增强了我们构建未来应用程序的方式——这是一个不容易实现的技巧。
