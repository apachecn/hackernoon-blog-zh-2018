# 来自 30 次面试的技术职业建议

> 原文：<https://medium.com/hackernoon/tech-career-advice-from-30-interviews-9c022acaad5>

![](img/f0e320845d1c8c429fc2a9fc8ae2bf81.png)

我刚刚结束了我的开发人员聊天系列的第一季，在那里我每 1-3 周采访一位不同的技术专家，询问他们的角色、职业和独特的经历。听到我的朋友们的声音真的很棒，我希望你们也喜欢看这些采访！

我们对这些人了解了很多，从他们如何开始，到如何进入角色，比如他们如何模仿打字。然而，在这篇文章中，我想总结一下面试中得到的职业建议。什么突出了？重复了什么？你现在能拿什么来改善自己的职业生涯？

# 做你自己

第一条建议是关于做你自己，做真实的自己，并坚持这样做。我采访过的很多人都曾在某个时候缺乏信心或动力，但现在，从另一方面来说，他们建议你自己去努力。偶尔也为自己抽出点时间。

*“天啊，我最近开始开玩笑说这是“带着白人的自信做事”(明目张胆地从一条推特上偷来的)。这很粗鲁吗？那有点粗鲁。不过这并没有错。无论如何，严肃地说，我被问了很多次，所以我做了一个关于它的大脑转储*[](https://gist.github.com/notwaldorf/4df04b5c9023aec6267bc7e66f284106)**。莫妮卡·丁库莱斯库**

**“做好自己。要成为一名工程师或从事技术工作，你不需要以某种特定的方式行事。* [*做你自己*](https://you.women2.com/programmatically-blonde-9054ebbbf1c7) *，把你多元真实的自己带到这个行业。我实际上认为* [*我古怪的品牌*](https://twitter.com/ChloeCondon/status/972669987520417793) *让我脱颖而出，让更多人听到我的声音。”*——[克洛伊·康登](https://www.samjarman.co.nz/blog/chloe)*

*这个绝妙的建议在采访中得到了回应:朱丽叶·布朗、T2、薇琪·布拉瑟尔、T4、拉弗尔·瓦格、约翰娜·鲁兹、克里斯蒂娜·巴拉姆和卡罗琳·索恩德。*

# *去拿吧！*

*这个建议围绕着开始朝着你想要的东西努力，无论是什么。恐惧、信心和更多的因素阻碍了开始——但是这些因素中有多少是真正的阻碍因素而不是想象的呢？我会让 JD Trask 完成这一部分。*

**“行动起来。这不仅限于你的软件职业。我遇到过很多想做点什么的人(升职、约喜欢的人出去、创业、翻新浴室——不胜枚举)。停止等待许可，停止阅读脸书的励志名言——基本上，停止胡闹，做该做的事情。没有人会给你东西。”* — JD Trask*

*同样告诉你去得到它的还有[阿塔·埃拉扬](https://www.samjarman.co.nz/blog/atta)和[西里斯·克里斯·克洛特](https://www.samjarman.co.nz/blog/cyris)……还有我！*

# *请求帮助*

*很多人在想问问题的时候会感到害怕，从长远来看，这对你是有害的。没什么好怕的，没有一个讲道理的人会因为问一个问题就给你负面的感觉。*

*我认为罗斯很好地总结了这一点，“寻求帮助不会让你变得软弱，这并不意味着你不值得呆在你现在的地方，也不意味着你没有能力”。*

*这个我怎么推荐都不为过的好建议也得到了凯利·沃恩、卡洛琳·索恩、T2、迈克·杰弗科特、T4、汉娜·格雷和萨米·皮奇利的响应。*

# *永远不要停止学习*

*应该清楚的是，在 2018 年，尤其是科技领域，唯一不变的是变化。新的工具、技术和更多的东西每天都在出现，而其他的则逐渐被淘汰。认为你已经完成了学习是职业生涯的克星，所以请继续学习。*

**“不可能什么都知道。总会有人知道的比你多。总会有一个专家比你做得更多。永远不要停止学习。永远不要停止尝试新事物。”杰斯·多德森**

**“不要规划你的职业生涯，世界瞬息万变。拥抱你周围的挑战，承担风险，看看你最终会在哪里。”劳拉·贝尔
Katrina Clokie:*这一观点可以用一句话来概括:“我认为最成功的人是那些总是愿意学习的人。”***

*这些聪明的人还告诉你永远不要停止学习:[莎拉·达扬](https://www.samjarman.co.nz/blog/sarah-dayan)、[比万·阿普斯](https://www.samjarman.co.nz/blog/bevan)、[劳拉·贝尔、](https://www.samjarman.co.nz/blog/laura-bell)、[克里斯蒂娜·巴兰](https://www.samjarman.co.nz/blog/kristina-balaam)、[和 YY·霍普](https://www.samjarman.co.nz/blog/andyy)、[乔·法比塞维奇](https://www.samjarman.co.nz/blog/mergesort)、[卡罗琳·索恩、](https://www.samjarman.co.nz/blog/carolynsaund)、[斯科特·汉塞尔曼](https://www.samjarman.co.nz/blog/shanselman)……你明白了吗？不要停下来！*

# *教授你所知道的*

*除了学习，你还应该教书。一个重要的作用，比如高级工程师，就是让更多的高级工程师。他们通过教学做到这一点，用他们的经验来帮助指导他们经验不足的同龄人。我喜欢凯利说的“如果你不知道，就去问”。如果你知道，就去教书。我们作为一个技术社区，通过分享而成长。我们一开始都不知道如何做我们今天做的事情。”这些话再正确不过了。*

*最后，来自 Sarah Dayan——“高级开发人员，接受你职位中的指导部分。成为大四学生不仅仅意味着更高的薪水和远离繁重的工作。如果你的公司不鼓励导师制度，或者在高风险的任务上过于依赖你，占用了你所有的时间，提醒他们教学的好处:工程团队内部更高的信任度，初级员工增长技能，以及短期、中期和长期的更好速度。”*

*这个建议也得到了莎拉·达扬、T2 和 YY·霍普以及 T4·汉娜·格雷的响应。*

# *积极学习核心技能*

*就像 JD 这么简洁的说的，*“编码是容易的部分，人是难的部分”*。致力于你的核心技能(又名[软技能](https://www.samjarman.co.nz/blog/core-skills))，与他人合作愉快，并通过网络结识他人将会给你的职业生涯带来很多好处。在新西兰，我们有一句美丽的谚语:*

> **他啊哈他的意思是你懂的**
> 
> **世界上最重要的是什么？**
> 
> **何唐卡塔，何唐卡塔，何唐卡塔**
> 
> **是人民，是人民，是人民**
> 
> *(毛利谚语)*

*这些至理名言也被几乎所有在 Dev Chats 中采访过的人所涵盖，所以其中一定有蹊跷。*

# *接受批评*

*批评和反馈是职业生涯的养料。他们会在你可能不知道的更多方面帮助你成长。我认为埃里卡·萨登总结得很好:“批评在成长和发展方面非常有价值。我认为，当人们受到批评时，他们自然会想为自己辩护，尽管很多批评可能真的很愚蠢(确实存在)，但我认为绝大多数都是人们在帮助你，拒绝这种帮助，不给予应有的考虑，不符合你的最佳利益。”更简洁地说，[奥林·肖](https://www.samjarman.co.nz/blog/aurynn)说:“*别再做混蛋了”。**

# *关注质量*

*最后，做出伟大的产品。在开发伟大的产品时，用你上面的换位思考技巧来考虑用户。正如迈克·杰弗科特所说，*“尽可能多地考虑终端用户对你在他们环境中所做的东西的体验。”*。更简单的说法是[尼克·帕芬](https://www.samjarman.co.nz/blog/nick)，*“努力让你的用户在使用你的软件时微笑。*”*

*这个建议得到了 [Atta Elayyan](https://www.samjarman.co.nz/blog/atta) 、 [Vicky Brasseur](https://www.samjarman.co.nz/blog/vicky) 和 [Sami Peachey](https://www.samjarman.co.nz/blog/sami) 的响应*

*虽然我在这里引用了一些话，但我相信我在第一季 Dev Chats 中交谈过的所有人都会同意这个建议。*

*我真的希望你能从中有所收获。点击任何一个名字，阅读完整的采访和更多内容。*
